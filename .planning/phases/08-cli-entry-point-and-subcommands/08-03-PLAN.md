    This is a migration of `src/index.ts` main() into a Commander command handler. The logic is nearly identical but structured differently:

    ```typescript
    import type { Command } from 'commander';
    import { loadConfig } from '../../config/loader.js';
    import { loadRole } from '../../config/role-loader.js';
    import { loadMeetingContext } from '../../meeting/loader.js';
    import { DeviceManager } from '../../devices/index.js';
    import { detectPlatform } from '../../platform/detect.js';
    import { createAudioCapture, createAudioOutput } from '../../audio/index.js';
    import type { AudioCapture, AudioOutput } from '../../audio/index.js';
    import { WslAudioRelayServer } from '../../audio/wsl2-relay-server.js';
    import { createVideoFeed, DEFAULT_PLACEHOLDER_PATH } from '../../video/index.js';
    import type { VideoFeed } from '../../video/index.js';
    import { GeminiProvider } from '../../ai/index.js';
    import { buildSystemPrompt } from '../../ai/index.js';
    import { TranscriptWriter } from '../../transcript/writer.js';
    import { OperatorAudioMonitor } from '../../monitor/operator-audio.js';
    import { AgentError, AISessionError } from '../../errors/index.js';

    interface StartOptions {
      config?: string;
      notes?: string;
      role?: string;
      verbose?: boolean;
    }

    export function registerStartCommand(program: Command): void {
      program
        .command('start')
        .description('Launch a meeting session')
        .option('-c, --config <path>', 'path to config.json')
        .option('-n, --notes <path>', 'path to meeting notes markdown file')
        .option('-r, --role <path>', 'path to persona/role file')
        .option('--verbose', 'enable verbose logging')
        .action(async (options: StartOptions) => {
          try {
            await startSession(options);
          } catch (err) {
            if (err instanceof AgentError) {
              console.error(`\nError: ${err.message}`);
              console.error(`Hint: ${err.hint}`);
              process.exit(err.exitCode);
            }
            console.error(`\nUnexpected error: ${(err as Error).message}`);
            process.exit(1);
          }
        });
    }
    ```

    The `startSession(options)` function contains the full startup sequence from src/index.ts main(), with these changes:

    1. **Config loading:** Use `loadConfig(options.config)` instead of `parseCliArgs` + `loadConfig(args.configPath)`
    2. **Role loading:** If `options.role`, call `loadRole(options.role)` and merge into config.persona (spread override: `config.persona = { ...config.persona, ...loadRole(options.role) }`)
    3. **Meeting notes:** Use `options.notes` instead of `args.meetingPath` (requirement CFG-02 renames --meeting to --notes)
    4. **AI provider:** Use `GeminiProvider` instead of `GeminiLiveSession` directly:
       ```typescript
       const provider = new GeminiProvider({
         apiKey,
         model: config.ai.model,
         systemPrompt,
         maxRetries: 5,
       });
       ```
       Wire events the same way (provider emits same events as GeminiLiveSession).
    5. **API key check:** If `!process.env.GEMINI_API_KEY`, throw `new AISessionError('GEMINI_API_KEY environment variable not set', 'Export GEMINI_API_KEY=your-key or add to .env file')` instead of `console.error` + `process.exit`
    6. **Error handling:** Device startup errors should throw DeviceError (DeviceManager already does). Audio pipeline errors should be wrapped in AudioPipelineError if not already.
    7. **Shutdown handler:** Same SIGINT/SIGTERM handler, but call `provider.disconnect()` instead of `session.disconnect()`
    8. **Verbose flag:** If `options.verbose`, log audio levels and reconnection events. If not verbose, suppress the per-chunk `[Capture] Audio level:` messages.

    Copy the entire startup logic from src/index.ts, applying the above changes. The flow is:
    - Platform detect
    - Load config (with --config path)
    - Load role (with --role path, merge into persona)
    - Load meeting notes (with --notes path)
    - Init transcript writer
    - DeviceManager startup
    - WSL2 relay server (if wsl2)
    - Audio capture + output (CRITICAL — must succeed)
    - AI provider connect (CRITICAL — must succeed)
    - Operator monitor (non-fatal)
    - Wire audio streams
    - Video feed (non-fatal)
    - Print startup banner
    - Register shutdown handlers
    - Keep-alive promise

    **IMPORTANT:** Do NOT call `process.exit()` in the main startup logic — throw AgentError subclasses instead. Only the `.action()` catch block calls `process.exit()`.

    The one exception is the shutdown handler which calls `process.exit(0)` after cleanup (this is intentional — clean exit after SIGINT).
