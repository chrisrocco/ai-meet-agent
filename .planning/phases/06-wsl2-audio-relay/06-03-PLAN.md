# Phase 6 Gap Closure — Fix Bridge Error Handling and Duplicate Log

## Objective

Fix two diagnosed bugs from UAT: (1) bridge ChildProcess errors crash the entire app because they are re-emitted as unhandled `'error'` events on the WslAudioRelayServer EventEmitter, and (2) the relay listening message is printed twice (once inside the class and once in index.ts). Both share one root cause area — the relay server's error propagation design.

## Context

@src/audio/wsl2-relay-server.ts — Lines 223-225 and 260-262: `this.emit('error', err)` re-emits ChildProcess errors. No listener exists on the instance in index.ts, so Node.js throws an unhandled error and crashes.

@src/index.ts — Line 58 duplicates the log already printed at wsl2-relay-server.ts line 33.

Two changes in two files:

**1. src/audio/wsl2-relay-server.ts — Replace bridge error re-emit with console.warn (lines 223-225 and 260-262)**

In `spawnCaptureBridge()`, replace:
```typescript
this.captureProc.on('error', (err) => {
  this.emit('error', err);
});
```
with:
```typescript
this.captureProc.on('error', (err) => {
  console.warn(`[AudioRelay:capture] Bridge error: ${err.message}`);
});
```

In `spawnOutputBridge()`, replace:
```typescript
this.outputProc.on('error', (err) => {
  this.emit('error', err);
});
```
with:
```typescript
this.outputProc.on('error', (err) => {
  console.warn(`[AudioRelay:output] Bridge error: ${err.message}`);
});
```

Rationale: Bridge errors (e.g., ffmpeg.exe not found) are expected on misconfigured systems. The bridge already auto-restarts on exit, so the error is recoverable. Logging as a warning keeps the operator informed without crashing the app. This matches the project's established non-fatal error handling pattern (see video feed, audio pipeline try/catch in index.ts).

Do NOT add a `relayServer.on('error', ...)` listener in index.ts — the class should not emit errors for non-fatal bridge failures in the first place.

**2. src/index.ts — Remove duplicate listening log (line 58)**

Remove this line:
```typescript
console.log(`[AudioRelay] TCP relay listening on port ${RELAY_PORT}`);
```

The identical message is already printed inside `WslAudioRelayServer.start()` at wsl2-relay-server.ts line 33, which uses the configured port (not the constant). The class log is the correct one to keep.

After removing the log line, also remove the `RELAY_PORT` import from the top of index.ts if it is no longer used anywhere else in the file (check before removing).
