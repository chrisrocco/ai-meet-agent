  <action>
Create `src/audio/wsl2-relay-server.ts` implementing `WslAudioRelayServer` class. This is the most complex file in the phase — follow the architecture patterns from the research document precisely.

**Class structure:**

```typescript
import { createServer, type Server, type Socket } from 'net';
import { spawn, type ChildProcess } from 'child_process';
import { EventEmitter } from 'events';
import { FrameReader, writeFrame, RELAY_PORT } from './wsl2-relay.js';
import type { Config } from '../config/schema.js';

export class WslAudioRelayServer extends EventEmitter {
  private server: Server | null = null;
  private captureClient: Socket | null = null;
  private outputClient: Socket | null = null;
  private captureProc: ChildProcess | null = null;
  private outputProc: ChildProcess | null = null;
  private outputReader: FrameReader | null = null;
  private stopped = false;
  private captureRestartTimer: ReturnType<typeof setTimeout> | null = null;
  private outputRestartTimer: ReturnType<typeof setTimeout> | null = null;

  constructor(private readonly config: Config) {
    super();
  }

  async start(): Promise<void> { ... }
  stop(): void { ... }
}
```

**1. `start()` method:**
- Call `startTcpServer()` first (returns Promise that resolves when listening)
- Then call `spawnCaptureBridge()` and `spawnOutputBridge()`
- Log: `[AudioRelay] TCP relay listening on port ${RELAY_PORT}`

**2. `startTcpServer()` method (private):**
- Create server with `createServer({ noDelay: true })`
- On 'connection': call `handleConnection(socket)`
- Listen on `127.0.0.1:RELAY_PORT` (use `this.config.audio.relayPort` if available, fall back to RELAY_PORT)
- Return Promise that resolves on 'listening', rejects on 'error'

**3. `handleConnection(socket)` method (private):**
- Set `socket.setNoDelay(true)`
- Create a FrameReader for the handshake
- Listen for 'data' on socket, feed to FrameReader
- When first complete frame arrives:
  - Remove the 'data' listener (the handshake listener)
  - Parse frame as JSON: `JSON.parse(frame.toString())`
  - If `handshake.type === 'capture'`:
    - If `this.captureClient` already exists, destroy old socket (new client replaces)
    - Set `this.captureClient = socket`
    - Log: `[AudioRelay] Capture client connected`
    - No further data listener needed — capture client only receives, it does not send audio
    - On 'close': set `this.captureClient = null`, log disconnect
  - If `handshake.type === 'output'`:
    - If `this.outputClient` already exists, destroy old socket
    - Set `this.outputClient = socket`
    - Create `this.outputReader = new FrameReader()`
    - Log: `[AudioRelay] Output client connected`
    - Add new 'data' listener that feeds data to `this.outputReader`, extracts frames, and writes raw PCM to `this.outputProc.stdin` (if bridge is running)
    - On 'close': set `this.outputClient = null`, `this.outputReader = null`, log disconnect
  - On 'error': log and destroy socket
- On socket 'error' during handshake: log and destroy

**4. `spawnCaptureBridge()` method (private):**
- Spawn ffmpeg.exe to capture from DirectShow device and output raw PCM to stdout
- Command: `spawn(this.config.wsl2.ffmpegPath, ['-f', 'dshow', '-audio_buffer_size', '50', '-i', 'audio=' + this.config.wsl2.captureDevice, '-f', 's16le', '-ar', '16000', '-ac', '1', 'pipe:1'], { stdio: ['ignore', 'pipe', 'pipe'], env: process.env })`
- On stdout 'data': if `this.captureClient` exists and is not destroyed, call `writeFrame(this.captureClient, pcm)` to forward the raw PCM chunk as a framed message to the capture client
- On stderr 'data': log first line only (ffmpeg prints a lot to stderr; log sparingly): `console.log('[AudioRelay:capture] ' + data.toString().split('\n')[0])`
- On 'exit': if `!this.stopped`, log restart, set `this.captureRestartTimer = setTimeout(() => this.spawnCaptureBridge(), 1000)`
- On 'error': emit('error', err)

**CRITICAL:** The bridge outputs RAW PCM bytes to stdout. The relay wraps these in length-prefixed frames via `writeFrame()` before sending to the capture TCP client. Do NOT use FrameReader on bridge stdout.

**5. `spawnOutputBridge()` method (private):**
- Spawn ffplay.exe to read raw PCM from stdin and play to Windows audio device
- Command: `spawn(this.config.wsl2.ffplayPath, ['-f', 's16le', '-ar', '16000', '-ac', '1', '-nodisp', '-autoexit', '-i', 'pipe:0'], { stdio: ['pipe', 'ignore', 'pipe'], env: process.env })`
  - Note: Do NOT use `-audio_device_index` if `outputDeviceIndex` is 0 (default device). Only add `-audio_device_index N` if N > 0.
- On stderr 'data': log first line sparingly
- On 'exit': if `!this.stopped`, log restart, set `this.outputRestartTimer = setTimeout(() => this.spawnOutputBridge(), 1000)`
- On 'error': emit('error', err)

**CRITICAL:** When forwarding output client audio to ffplay, the FrameReader extracts the raw PCM payload (stripping the 4-byte length header). Write ONLY the raw PCM payload to ffplay's stdin. ffplay expects raw PCM, NOT framed data.

**6. `stop()` method:**
- Set `this.stopped = true`
- Clear both restart timers
- Close `this.server` (server.close())
- Destroy `this.captureClient` and `this.outputClient` sockets
- Kill bridge processes:
  - For each bridge proc: if proc and !proc.killed, kill it
  - Use `spawn('powershell.exe', ['-Command', 'taskkill /F /T /PID ' + proc.pid])` to kill the Windows process tree (matching the Wsl2VideoFeed pattern — simple proc.kill() may not kill the Windows-side process)
- Null out all references

**7. Private helper `killBridge(proc: ChildProcess | null)`:**
- If proc is null or proc.killed, return
- Try `spawn('powershell.exe', ['-Command', 'taskkill /F /T /PID ' + proc.pid])` to kill Windows process tree
- Also call `proc.kill('SIGTERM')` as fallback for the WSL2-side process

**Event emissions:**
- `'error'` with Error payload for bridge spawn failures
- No other events needed — logging is sufficient for debugging

**Important implementation notes:**
- Always pass `{ env: process.env }` to child_process.spawn() for bridge processes — ensures WSL2 interop ($WSL_INTEROP) is inherited
- Use `socket.setNoDelay(true)` on EVERY accepted socket
- When output client disconnects and reconnects, the old `outputReader` must be replaced with a fresh one (reset state)
- If bridge process is not running when client sends data, just drop the audio silently (log at debug level if desired)
- If capture client is not connected when bridge produces audio, just drop the audio silently
