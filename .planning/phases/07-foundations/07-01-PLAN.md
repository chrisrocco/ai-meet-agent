<feature>
  <name>AgentError class hierarchy</name>
  <files>src/errors/index.ts, src/errors/index.test.ts</files>
  <behavior>
    - AgentError extends Error and carries .message (string), .hint (string), .exitCode (number)
    - AgentError.name returns 'AgentError' (not 'Error')
    - new AgentError('msg', 'hint') defaults exitCode to 1
    - new AgentError('msg', 'hint', 5) sets exitCode to 5
    - ConfigError extends AgentError with exitCode=2, default hint='Check your config.json'
    - DeviceError extends AgentError with exitCode=3, default hint='Run "bash scripts/setup.sh" to install prerequisites'
    - AISessionError extends AgentError with exitCode=4, default hint='Check GEMINI_API_KEY and network connection'
    - AudioPipelineError extends AgentError with exitCode=5, default hint='Check PulseAudio/PipeWire setup'
    - Each subclass .name returns its own class name (e.g. 'ConfigError')
    - instanceof checks work: `new ConfigError('x') instanceof AgentError` is true
    - Custom hint overrides default: new ConfigError('msg', 'custom hint') uses 'custom hint'
  </behavior>
  <implementation>
    Create a flat hierarchy — one level of subclasses. Each subclass:
    - Calls super(message, hint ?? defaultHint, exitCode)
    - Base class sets this.name = this.constructor.name in constructor
    - All classes exported from src/errors/index.ts

    Error subclasses and their exit codes:
    | Class | exitCode | Default hint |
    |-------|----------|-------------|
    | AgentError | 1 | (required param) |
    | ConfigError | 2 | 'Check your config.json' |
    | DeviceError | 3 | 'Run "bash scripts/setup.sh" to install prerequisites' |
    | AISessionError | 4 | 'Check GEMINI_API_KEY and network connection' |
    | AudioPipelineError | 5 | 'Check PulseAudio/PipeWire setup' |

    JSDoc on every class documenting when to throw it and what exitCode it carries.
  </implementation>
</feature>
