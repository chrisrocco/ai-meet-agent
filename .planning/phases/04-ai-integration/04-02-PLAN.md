<task type="auto" tdd="true">
  <name>Task 2: Build GeminiLiveSession wrapper</name>
  <files>src/ai/session.ts, src/ai/session.test.ts, src/ai/index.ts</files>
  <behavior>
    - Test: GeminiLiveSession.connect() calls ai.live.connect with correct config (model, responseModalities, systemInstruction)
    - Test: GeminiLiveSession.sendAudio() base64-encodes PCM and sends via session.sendRealtimeInput
    - Test: GeminiLiveSession handles serverContent audio by decoding base64, downsampling 24k->16k, and emitting 'audio' event
    - Test: GeminiLiveSession reconnects on transient error with exponential backoff
    - Test: GeminiLiveSession does NOT reconnect on permanent error (401/403/429)
    - Test: GeminiLiveSession stops retrying after maxRetries (5) and emits 'error'
    - Test: GeminiLiveSession.disconnect() closes session cleanly
    - Test: GeminiLiveSession state transitions: disconnected -> connecting -> connected -> disconnected
  </behavior>
  <action>
  1. Create `src/ai/session.ts`:
     - Import `{ GoogleGenAI, Modality }` from '@google/genai'
     - Import `EventEmitter` from 'events'
     - Import types from './types.js', `downsample24to16` from './audio-converter.js', `LatencyTracker` from './latency.js'

     - Export class `GeminiLiveSession extends EventEmitter`:
       - Constructor takes: `config: { apiKey: string, model: string, systemPrompt: string, maxRetries?: number }`
       - Private fields:
         - `ai: GoogleGenAI` — initialized with apiKey
         - `session: any | null` — the live session object
         - `state: GeminiSessionState` — starts as 'disconnected'
         - `retryCount: number` — current retry attempt
         - `maxRetries: number` — default 5
         - `latency: LatencyTracker` — internal tracker instance
         - `intentionalDisconnect: boolean` — prevents reconnect on manual disconnect

       - `async connect(): Promise<void>`:
         - Set state to 'connecting'
         - Call `this.ai.live.connect()` with:
           - model: `this.config.model`
           - config.responseModalities: `[Modality.AUDIO]`
           - config.systemInstruction: `{ parts: [{ text: this.config.systemPrompt }] }`
         - Set up callbacks:
           - onopen: set state 'connected', reset retryCount, emit 'connected', start latency summary timer
           - onmessage: handle serverContent (extract audio parts, base64 decode, downsample, emit 'audio')
           - onerror: classify error, if transient -> reconnect, if permanent -> emit 'error'
           - onclose: if not intentional disconnect -> attempt reconnect

       - `sendAudio(pcmChunk: Buffer): void`:
         - If not connected, silently drop (per user decision: mic goes silent during reconnect)
         - Base64 encode the chunk
         - Call `session.sendRealtimeInput({ audio: { data: base64, mimeType: 'audio/pcm;rate=16000' } })`
         - Call `this.latency.markSent()`

       - `async disconnect(): Promise<void>`:
         - Set intentionalDisconnect = true
         - Close session
         - Set state 'disconnected', emit 'disconnected'
         - Stop latency summary timer

       - Private `async reconnect(): Promise<void>`:
         - Set state 'reconnecting'
         - Loop up to maxRetries:
           - Exponential backoff: `Math.min(1000 * 2^(attempt-1), 30000)` ms
           - Log: `[AI] Reconnecting in {delay}ms (attempt {N}/{max})`
           - await delay
           - Try connect()
           - On success: return
           - On transient error: continue loop
           - On permanent error: emit 'error', return
         - If all retries exhausted: log `[AI] Max retries exhausted`, emit 'error' with descriptive message

       - Private `isTransientError(error: Error): boolean`:
         - Check error message for permanent indicators: '401', '403', '429', 'invalid', 'unauthorized', 'quota', 'forbidden'
         - If any match: return false (permanent)
         - Otherwise: return true (transient — network, timeout, reset)

       - Private `handleMessage(msg: any): void`:
         - Check for serverContent?.modelTurn?.parts
         - For each part with inlineData: decode base64 to Buffer (24kHz PCM), downsample to 16kHz, emit 'audio' event with 16kHz Buffer
         - On first audio response after a send: call latency.markReceived()

       - Getter `getState(): GeminiSessionState`
       - Getter `getLatencyStats(): LatencyStats` — delegates to latency tracker

  2. Create `src/ai/session.test.ts`:
     - Mock `@google/genai` to avoid real API calls
     - Test all behaviors listed above using mock session objects
     - For reconnection tests: simulate onerror/onclose callbacks with transient vs permanent errors
     - Verify exponential backoff timing using fake timers

  3. Update `src/ai/index.ts` to re-export from './session.js' and './latency.js'
