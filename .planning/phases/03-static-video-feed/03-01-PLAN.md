**1. Create `src/video/wsl2-feed.ts`** — Wsl2VideoFeed class:
- Extends EventEmitter, implements VideoFeed
- Constructor takes `port: number` (default from config)
- `start(imagePath)`:
  1. Start HTTP server on `0.0.0.0:port` serving `multipart/x-mixed-replace` MJPEG
  2. Convert imagePath to Windows path via `execSync('wslpath -w "..."').toString().trim()`
  3. Spawn `powershell.exe` with args: `['-Command', 'ffmpeg.exe -loop 1 -re -i "{winPath}" -vf "scale=1280:720:force_original_aspect_ratio=decrease,pad=1280:720:(ow-iw)/2:(oh-ih)/2,setsar=1" -f mjpeg -r 15 pipe:1']`
  4. Use broadcast pattern for HTTP clients (NOT `.pipe()` which only serves one client):
     - Track active `ServerResponse` objects in a `Set<ServerResponse>`
     - On ffmpeg stdout 'data', write to ALL active responses
     - On response 'close', remove from set
  5. On ffmpeg 'error': emit('error', err)
  6. On ffmpeg 'exit': if (!stopped) emit('restarting'), setTimeout 1s, restart
- `stop()`:
  1. Set stopped=true, clear restartTimer
  2. Kill Windows process tree: `spawn('powershell.exe', ['-Command', 'taskkill /F /T /PID ' + pid])`
  3. Close HTTP server, clear client set
- Note: Use `-f mjpeg` (not `-f mpjpeg`) to output raw MJPEG frames to stdout. The HTTP server wraps each frame with multipart boundaries manually. This avoids dependency on `-mpjpeg_boundary_tag` flag availability (MEDIUM confidence per research).
  Alternative simpler approach: Use `-f image2pipe -vcodec mjpeg pipe:1` which outputs raw JPEG frames. The HTTP server adds `--boundary\r\nContent-Type: image/jpeg\r\n\r\n` before each frame.
