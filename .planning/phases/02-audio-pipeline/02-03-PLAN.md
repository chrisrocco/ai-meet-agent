<objective>
Implement WSL2 audio bridge: TCP relay protocol, WSL2-side capture/output clients, and Windows-side relay server.

Purpose: WSL2 cannot run parec/pacat directly (no PulseAudio server). Instead, audio is relayed over TCP between Node.js in WSL2 and a relay server on Windows. The relay server runs as a Node.js script on the Windows side, auto-launched via powershell.exe from WSL2. This plan creates the bidirectional bridge.

Output: TCP framing utilities, WSL2 capture/output classes, Windows relay server script
</objective>

<task type="auto">
  <name>Task 2: WSL2 capture/output clients and Windows relay server</name>
  <files>src/audio/wsl2-capture.ts, src/audio/wsl2-output.ts, scripts/wsl2-audio-relay.js</files>
  <action>
    **Wsl2AudioCapture (src/audio/wsl2-capture.ts):**
    - Extends EventEmitter, implements AudioCapture
    - Constructor takes sinkName (for protocol identification) and relay port (default RELAY_PORT)
    - `start()`: Connect to localhost:RELAY_PORT via net.connect(). Send a handshake frame: `writeFrame(socket, Buffer.from(JSON.stringify({ type: 'capture', sink: sinkName })))`. Create a PassThrough stream. On socket 'data', feed to FrameReader, push each extracted frame (PCM data) to the PassThrough. Emit 'level' events with RMS. Return the PassThrough as Readable.
    - `stop()`: Destroy socket, end PassThrough
    - On socket 'error' or 'close': emit 'reconnecting', attempt reconnect after 1s
    - On socket 'error': emit 'error' event

    **Wsl2AudioOutput (src/audio/wsl2-output.ts):**
    - Extends EventEmitter, implements AudioOutput
    - Constructor takes sinkName and relay port
    - `start()`: Connect to localhost:RELAY_PORT. Send handshake frame: `writeFrame(socket, Buffer.from(JSON.stringify({ type: 'output', sink: sinkName })))`. Create a Transform stream that on each chunk: `writeFrame(socket, chunk)`, emit 'level' with RMS. Return the Transform as Writable.
    - `stop()`: End the transform, destroy socket

    **Windows relay server (scripts/wsl2-audio-relay.js):**
    - Standalone Node.js script (plain JS, no TypeScript — runs via node.exe on Windows)
    - Creates TCP server on RELAY_PORT (19876)
    - On connection: read first frame (handshake JSON with type + sink name)
    - If type='capture': spawn parec on Windows PulseAudio (if available) or read from VB-Cable device. For initial implementation: spawn `parec` with the sink name, pipe stdout data as frames to the TCP client. NOTE: This is a best-effort implementation — actual Windows audio routing depends on the user's setup.
    - If type='output': spawn pacat with the sink name, pipe incoming TCP frames to pacat stdin
    - Handle disconnect: kill subprocess, clean up
    - Log all connections and disconnections to stderr
    - Add `#!/usr/bin/env node` shebang for direct execution
    - Include clear comments about Windows audio requirements (VB-Cable, PulseAudio for Windows, etc.)
