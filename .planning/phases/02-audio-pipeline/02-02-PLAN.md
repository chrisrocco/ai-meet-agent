<objective>
Implement native Linux audio capture (parec) and output (pacat) classes.

Purpose: These are the core audio I/O classes for native Linux. NativeAudioCapture reads from PulseAudio sink monitor via parec subprocess, producing a Node.js Readable stream. NativeAudioOutput writes PCM to the virtual mic sink via pacat subprocess stdin. Both emit RMS level events and handle subprocess lifecycle.

Output: src/audio/capture.ts, src/audio/output.ts, and their test files

  <action>
    Create NativeAudioCapture class extending EventEmitter, implementing AudioCapture interface:
    - Constructor takes `sinkName: string` (e.g., 'ai_meet_sink') and optional format config
    - `start()`: spawn `parec` with args `['--device', '{sinkName}.monitor', '--format', 's16le', '--rate', '16000', '--channels', '1', '--latency-msec', '20']`. Pipe stdout data through an RMS calculator that emits 'level' events every ~8000 bytes (250ms at 16kHz/16-bit/mono = 32KB/s, so 8KB = 250ms). Return proc.stdout as the Readable.
    - `stop()`: Set `this.stopped = true` (prevent reconnect), then `proc.kill('SIGTERM')`, null out proc reference
    - On subprocess 'exit': if `!this.stopped`, emit 'reconnecting', wait 1 second, call start() again
    - On subprocess stderr 'data': log to console.error with `[parec]` prefix
    - On subprocess 'error': emit 'error' event

    For tests: Use node:test with mock/spy on child_process.spawn. Create a fake ChildProcess that emits events. Verify spawn args, event emissions, reconnect behavior.
  </action>
