  <behavior>
    ## Types (src/audio/types.ts)
    Define the following (no tests needed — pure type definitions):

    - `AUDIO_FORMAT` constant object: `{ sampleRate: 16000, bitDepth: 16, channels: 1, encoding: 's16le' as const }`
    - `AudioConfig` type: `{ sinkName: string; format: typeof AUDIO_FORMAT }`
    - `AudioCapture` interface extending EventEmitter:
      - `start(): Readable` — returns the PCM stream
      - `stop(): void` — kills subprocess, prevents reconnect
      - Events: `'reconnecting'`, `'error'`, `'level'` (with RMS number payload)
    - `AudioOutput` interface extending EventEmitter:
      - `start(): Writable` — returns the PCM writable stream
      - `stop(): void` — ends stdin, kills subprocess
      - Events: `'error'`, `'level'` (with RMS number payload)
    - Export all types

    ## PCM Utilities (src/audio/pcm-utils.ts, src/audio/pcm-utils.test.ts)
    TDD the following behaviors:

    - `computeRms(buffer)`: Given a Buffer of s16le PCM samples, returns the RMS value as a number
      - Test: silence buffer (all zeros) → returns 0
      - Test: max amplitude buffer (all 32767) → returns 32767
      - Test: known sine wave samples → returns expected RMS within 1% tolerance
      - Test: empty buffer → returns 0
      - Test: odd-length buffer (not aligned to 2 bytes) → handles gracefully (ignores trailing byte)

    - `computeRmsNormalized(buffer)`: Returns RMS normalized to 0-1 range (divide by 32768)
      - Test: silence → 0
      - Test: max amplitude → approximately 1.0
  </behavior>
