## Integration Issues

### Critical

**WSL2 relay port config not threaded to TCP clients**
- `WslAudioRelayServer` reads `config.audio.relayPort` (default 19876)
- `Wsl2AudioCapture` and `Wsl2AudioOutput` hard-code `RELAY_PORT` constant
- If config overrides port, relay listens on custom port but clients connect to 19876
- Silent failure — no error, just no audio
- Affects: PLAT-02, AUDI-01

### Medium

**Wsl2VideoFeed spawns Linux ffmpeg instead of ffmpeg.exe**
- `src/video/wsl2-feed.ts` spawns `'ffmpeg'` (Linux binary)
- On WSL2 without native ffmpeg installed, video feed silently fails
- WslAudioRelayServer correctly uses configurable `ffmpegPath` — video should follow same pattern
- Affects: VDEV-03, PLAT-02

**Async session.disconnect() called without await**
- `src/index.ts` shutdown handler is synchronous, calls `session.disconnect()` (async) without await
- WebSocket may not close cleanly before `process.exit(0)`
- Affects: AUDI-05
